## 🌐 Cloud Sync Features

- ✅ **Active Alerts**: Live synchronization of all active alarms

- ✅ **Historical Data**: Cloud-based alert history

### Offline Capabilities
- ✅ **Offline Persistence**: Works without internet connection
- ✅ **Automatic Reconnection**: Seamless sync when back online

### Push Notifications (FCM)
- ✅ **Critical Alerts**: Instant push for critical alarms
- ✅ **Background Notifications**: Receive alerts when app is closed
- ✅ **Custom Sounds**: Alert-specific notification sounds
- ✅ **Multi-device Support**: Sync across Windows, Android, Web

### Heartbeat Monitoring
- ✅ **Client Heartbeat**: Every 30 seconds
- ✅ **Service Status**: Monitor all components
- ✅ **Connection State**: Real-time online/offline detection

## 📦 Deployment Steps

### Complete Setup Script

Run the automated setup script:

```powershell
.\setup_firebase_complete.ps1
```

This script will:
1. ✅ Verify Firebase CLI installation
2. ✅ Authenticate with Firebase
3. ✅ Deploy Firestore security rules
4. ✅ Deploy Firestore indexes
5. ✅ Deploy Storage rules
6. ✅ Configure FlutterFire for all platforms

### Manual Deployment (Alternative)

#### 1. Deploy Firestore Rules
```powershell
firebase deploy --only firestore:rules
```

#### 2. Deploy Firestore Indexes
```powershell
firebase deploy --only firestore:indexes
```

#### 3. Deploy Storage Rules
```powershell
firebase deploy --only storage
```

#### 4. Configure FlutterFire
```powershell
flutterfire configure `
  --project=scadadataserver `
  --platforms=android,windows,web `
  --android-package-name=com.industrial.scada_alarm_client `
  --windows-app-id=com.industrial.scada_alarm_client `
  --out=lib/firebase_options.dart `
  --yes
```

## 🔐 Security Rules

### Firestore Rules (`firestore.rules`)

- **Public Read**: Alerts can be read by anyone (for monitoring displays)
- **Operator Write**: Only operators can create/update alerts
- **Admin Control**: Admins have full access
- **User Privacy**: Users can only read their own data

### Storage Rules (`storage.rules`)

- **File Size Limits**: 
  - Alert attachments: 10 MB
  - Reports: 50 MB
  - Profile images: 5 MB
- **Content Type Validation**: Only images and PDFs allowed
- **User-based Access**: Users can only access their own files

## 🐛 Troubleshooting

### Firebase Initialization Fails

**Symptoms:**
```
⚠️ Firebase initialization failed: [error]
```

**Solutions:**
1. Verify `firebase_options.dart` exists
2. Check internet connection
3. Validate project ID in Firebase Console
4. Run `flutterfire configure` again

### Sync Service Not Working

**Symptoms:**
- No real-time updates
- Offline mode not detected

**Solutions:**
1. Check `connectivity_plus` package is installed
2. Verify Firestore rules allow read access
3. Check console for network errors
4. Enable Firebase debug logging

### Push Notifications Not Received

**Symptoms:**
- No notifications appear

**Solutions:**
1. Verify FCM is enabled in Firebase Console
2. Check Windows notification permissions
3. Verify FCM token is saved to Firestore
4. Check `firebase-messaging-sw.js` for web platform

### Data Not Syncing

**Symptoms:**
- Changes in Firebase Console don't appear in app

**Solutions:**
1. Check Firestore indexes are deployed
2. Verify collection names match exactly
3. Check Firestore security rules
4. Enable offline persistence in Firestore
